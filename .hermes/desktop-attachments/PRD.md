# env PRD

상태: app-level source of truth.

## 한 줄 정의

env는 공부를 시작하기 전 여러 설정을 선택하며 집중력을 소모하고, 여러 도구를 오가는 동안 다른 길로 새는 문제를 해결한다.

## 해결하려는 문제

공부를 시작하지 못하는 문제에는 여러 원인이 있다. 피곤함, 불안, 회피, 무엇부터 할지 모름, 낮은 에너지, ADHD 또는 executive dysfunction 같은 원인도 있을 수 있다.

env가 해결하려는 문제는 그 전체가 아니다.

env가 해결하려는 정확한 문제는 다음 두 가지다.

> 공부 시작 전에 타이머, Screen Time, 사운드 등을 선택하면서 집중력이 무의식적으로 먼저 소모되는 것.

그리고:

> 이 도구들을 각각 찾고 설정하고 전환하는 동안, 공부가 아닌 다른 길로 새는 것.

이 문제에서 폰은 완전히 없애야 할 대상이 아니다. 공부 자료, PDF, 브라우저, 사전, 강의, 인증, 연락, 타이머, 앱 차단, 사운드처럼 필요한 접촉도 있다.

문제는 공부를 준비하기 위한 선택과 앱 전환이 집중 시작 전에 몰려 있다는 점이다. 사용자는 타이머 앱, Screen Time 설정, 사운드 앱 등을 차례로 다루면서 아직 공부를 시작하지도 않았는데 집중력을 선택에 사용한다. 동시에 각 도구를 찾고 오가는 과정에서 SNS, YouTube, 메시지, 브라우저, 쇼츠, 릴스, 게임으로 새기 쉽다.

따라서 env가 줄여야 하는 것은 폰 접촉 전체가 아니라:

> 공부 전에 발생하는 선택과 설정, 그리고 여러 도구 사이의 앱 전환.

## 문제의 두 축

env가 해결해야 하는 병목은 두 축으로 나뉜다.

### 1. 인지 병목

사용자가 공부를 시작하려는 순간 `Timer`, `Screen Time`, `Allowed Apps`, `Sound`, `Break`를 다시 고르면, 선택 하나하나가 집중력을 무의식적으로 소모한다.

이 병목은 설정 시간이 길다는 문제가 아니다. 다른 길로 새지 않고 모든 설정을 완료하더라도, 공부에 써야 할 인지 자원을 시작 전에 이미 사용했다는 것이 문제다.

해결 방향:

- 공부 환경은 session 시작 전에 미리 세팅되어 있어야 한다.
- `Focus Env`는 Timer, Screen Time, Allowed Apps, Sound, Break를 저장한 사전 설정이다.
- 사용자는 공부 시작 순간에 값을 다시 선택하지 않고, 이미 만들어둔 env를 실행해야 한다.
- 목표는 설정 선택에 집중력을 쓰지 않은 상태로 공부를 시작하게 하는 것이다.

### 2. 도달 병목

공부 환경을 준비하려면 타이머 앱, Screen Time 설정, 사운드 앱처럼 서로 다른 도구에 각각 도달해야 한다. 이 병목은 env 앱이나 특정 env를 찾기 어렵다는 뜻이 아니다.

현재 시작 경로는 다음처럼 길어질 수 있다.

```text
공부 의지 발생
→ 잠금 해제
→ 타이머 앱을 찾아 설정
→ Screen Time 설정으로 이동
→ 사운드 앱을 찾아 음악 선택
→ 이동 중 SNS, YouTube, 메시지, 브라우저에 노출
→ 시작
```

이처럼 여러 도구를 찾고 설정하고 전환하는 과정 자체가 공부가 아닌 다른 길로 새는 경로를 만든다.

해결 방향:

- 타이머, Screen Time, 사운드 등을 각각 찾아다니지 않아도 하나의 env가 필요한 환경을 함께 실행해야 한다.
- App Home에서는 준비된 env를 바로 boot할 수 있어야 한다.
- Home Screen widget과 Lock Screen widget은 이 통합된 시작 경로를 더 짧게 만드는 보조 entry point다.
- 위젯을 누르면 여러 도구를 거치지 않고 해당 env의 boot가 바로 시작되어야 한다.

### 두 축의 연결

`미리 세팅`은 공부 전에 선택으로 소모되는 집중력을 보존한다.

`하나로 통합된 실행`은 여러 도구를 오가는 동안 다른 길로 새는 것을 막는다.

두 축은 함께 작동해야 한다. 값이 미리 세팅되어 있지 않으면 env 안에서 다시 선택하며 집중력을 소모한다. 실행이 하나로 통합되어 있지 않으면 사용자는 여러 도구를 오가다가 다른 길로 샐 수 있다.

## 무엇을 해결하는가

env는 집중 기능을 편하게 묶어주는 앱이 아니다.

`Timer`, `Shield`, `Allowed Apps`, `Sound`, `Break`를 묶는 이유는 편의성이 아니다. 그 기능들을 하나씩 켜기 위해 폰을 만지는 행위 자체가 공부 시작을 방해할 수 있기 때문이다.

env가 해결하는 것은 다음이다.

- 공부 전에 타이머, Screen Time, 사운드 등을 선택하며 집중력을 먼저 소모하는 문제
- 타이머 앱, Screen Time 설정, 사운드 앱을 오가는 동안 딴짓 앱으로 새는 문제
- 폰이 공부에 필요하지만 동시에 가장 큰 방해물이 되는 문제
- 공부 시작 전에 선택과 설정이 많아져 집중력이 온전한 상태로 시작하지 못하는 문제
- 공부 의지가 생겼지만 준비 도구를 다루는 동안 시작이 지연되거나 실패하는 문제

env가 해결하지 않는 것은 다음이다.

- 공부할 의도 자체를 만들어주는 것
- 모든 집중력 문제를 해결하는 것
- 피로, 불안, 우울, 회피, 건강 문제를 해결하는 것
- ADHD 치료 또는 clinical 효과를 제공하는 것
- 폰을 완전히 못 쓰게 하는 parental control

## 사용자 언어

Desk research에서 반복적으로 발견한 표현이다. 이는 수요 신호이며, 아직 사용자 인터뷰나 지불 의향으로 검증된 것은 아니다.

| 사용자 표현 | 해석 |
| --- | --- |
| "My brain says study but my thumb says scroll" | 공부 의도는 있지만 손이 폰으로 간다. |
| "I sit down to study and 10 mins later I'm on my phone" | 공부하려고 앉았지만 시작 또는 유지로 전환하지 못한다. |
| "Every time I sit down, I end up reaching for my phone" | 공부 시작 순간마다 폰으로 새는 패턴이 있다. |
| "Whenever I go to study I always end up being on phone" | 공부하려고 할 때마다 결국 폰을 보게 된다. |
| "Pre-work period... 30mins - 1 hour of going on my phone" | 일이나 공부 전 폰 시간이 습관처럼 붙어 있다. |
| "When I need my phone to study I get distracted" | 폰이 공부에 필요하지만 그 자체가 위험한 도구다. |
| "공부해야 하는데 폰만 붙잡고" | 공부 의지는 있지만 폰에서 벗어나지 못한다. |
| "공부하려고 앉았는데, 어느새 스마트폰" | 공부 의도 이후 폰 접촉으로 이탈한다. |
| "할것도없는데 뭘그리 만지게되는지" | 목적 없는 폰 접촉이 반복된다. |
| "영어공부 해야되는데 계속 폰질만 하니까 나아지는게없넴" | 해야 할 공부가 있지만 폰질이 시작을 막는다. |

## 문제 구조

```text
공부 의지 발생
→ 타이머, Screen Time, 사운드 등을 선택
├─ 선택마다 집중력이 소모됨
└─ 여러 도구를 오가며 딴짓 경로에 노출됨
   → SNS, YouTube, 메시지, 브라우저, 게임으로 샘
→ 집중력이 소모된 상태로 시작하거나 시작 자체가 지연·실패함
```

env가 개입하는 지점은 “공부 의지” 자체가 아니다.

env가 개입하는 지점은:

> 공부 의지가 생긴 뒤, 준비를 위한 선택과 도구 전환이 실제 공부 시작을 방해하는 구간.

## 제품이 제공해야 할 결과

사용자가 공부하려고 마음먹은 순간, 폰을 오래 만지지 않고 바로 공부 상태로 들어가야 한다.

이를 위해 env는:

- 미리 만든 `Focus Env`를 한 번에 켠다.
- 공부 시작 시점에 설정을 다시 고르게 하지 않는다.
- App Home, Home Screen widget, Lock Screen widget에서 특정 env boot로 바로 들어가게 한다.
- 공부에 필요한 앱만 남기고 딴짓 앱 접근을 줄인다.
- Timer, Shield, Allowed Apps, Sound를 session 시작 전에 이미 정해진 값으로 실행한다.
- 사용자가 앱 안에서도 오래 탐색하거나 설정을 만지지 않게 한다.

## P0 기능

P0는 `인지 병목`과 `도달 병목`을 줄이는 최소 기능으로 제한한다.

### Focus Env preset

사용자는 공부 환경을 미리 만들어둘 수 있어야 한다.

- Env는 Timer, Shield, Allowed Apps, Sound, Break를 저장한다.
- Session 시작 시점에는 이 값을 다시 고르지 않는다.
- Env를 실행하면 저장된 설정으로 바로 focus state에 들어간다.
- Session이 시작되면 Env 설정은 고정된다. 설정을 바꾸려면 session을 끝내고 Env를 수정한 뒤 다시 시작한다.

### App Home quick boot

앱 안에서는 Home이 설정 화면이 아니라 env launcher처럼 동작해야 한다.

- Home에서 env를 선택 후 boot 를 누르면 해당 env boot가 바로 시작된다.
- Env 선택 이후 별도 시작 확인 단계를 두지 않는다.

### Home Screen widget

사용자는 자주 쓰는 env를 Home Screen widget에 둘 수 있어야 한다.

- Widget은 특정 env에 연결된다.
- Widget을 누르면 앱 홈이 아니라 해당 env boot로 바로 들어간다.

### Lock Screen widget

사용자는 잠금화면에서 자주 쓰는 env에 접근할 수 있어야 한다.

- Lock Screen widget은 특정 env에 연결된다.
- Widget을 누르면 env 선택 화면이 아니라 해당 env boot가 바로 시작된다.

## 온보딩 권한

Onboarding은 첫 Focus Env를 만드는 과정이므로, 사용자가 확정한 env 값에 필요한 권한을 해당 step의 맥락에서 확인할 수 있다.

- `nextPhase`에서 notification-style 진행을 선택하면 notification permission을 확인하고, 권한이 없으면 요청한다.
- Notification permission이 없으면 해당 step에 머무르고 `nextPhase` 선택값은 notification을 쓰지 않는 `auto`로 되돌린다.
- `screenTime`에서 app blocking을 `on`으로 확정하면 Screen Time / Family Controls permission을 확인하고, 권한이 없으면 요청한다.
- Screen Time / Family Controls permission이 승인되지 않으면 해당 step에 머무르고 `screenTime` 선택값은 `off`로 되돌린다.
- 권한 요청은 실제 env 값 확정 맥락에서만 발생해야 하며, 온보딩 시작 직후 선제적으로 요청하지 않는다.

## 포지셔닝 문장

사용할 수 있는 방향:

- 공부해야 하는데 폰만 붙잡고 있다면.
- 공부 시작 전에 새지 않게.
- 공부하려고 폰을 켠 순간, 바로 focus state로.
- 공부 의지가 살아있을 때, 한 번에 환경을 켜세요.
- 공부에 필요한 앱만 남기고 바로 시작.

피해야 할 방향:

- 올인원 집중 앱
- 타이머, 앱 차단, 사운드를 한곳에 모은 앱
- 집중력을 올려주는 앱
- 공부 동기부여 앱
- ADHD 치료 앱
- Screen Time 대체 앱
