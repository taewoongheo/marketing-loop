# Message MSG-001 v2: The setup is not the work

**Status:** testing
**Version:** 2

## Target user and situation

A college student has already decided what to study but is still choosing timer, Screen Time, allowed-app, sound, break, or cycle settings before touching the actual work.

## Current belief

Choosing the focus setup is harmless preparation. If starting still feels difficult afterward, the problem must be a lack of discipline or focus.

## Current behavior

The student rebuilds several familiar setup values at the moment of starting while the assignment, reading, or first problem waits.

## Intended belief shift

The setup is not the work. Repeating familiar setup choices may use attention the student intended to bring to the first real study action.

This is a message hypothesis, not an established scientific claim.

## Core perspective

Decide recurring setup values before the study session. At the moment of starting, run the prepared environment instead of choosing its parts again.

Different kinds of study can use different saved environments; reuse does not require one rigid setup for everything.

## Persuasion mechanism

1. Name the concrete timer, Screen Time, sound, blocked-app, allowed-app, and break choices still happening.
2. Keep the intended assignment, reading, paragraph, or problem visibly waiting while those choices are made.
3. Separate configuring how to focus from doing the first part of the chosen work.
4. Show the same values already decided inside a reusable environment.
5. End on the first meaningful study action rather than treating configuration as the achievement.

## Reader resistance to address

- “A focus app might become one more thing to configure.” Show one prepared environment being reused rather than rebuilt at every start.
- “A saved setup might be too rigid.” Show different recurring environments holding different prepared settings.
- “These choices are tiny.” Make the repeated choices visible without claiming a measured cognitive cost.
- “Maybe I just need more discipline.” Describe the environmental friction without claiming discipline or motivation no longer matters.

## Concrete scenes and language

Prefer observable scenes such as:

- choosing a timer while the first page waits;
- deciding on sound before opening the assignment;
- adjusting blocked and allowed apps before beginning the first problem;
- rebuilding the same timer, break, and sound setup before every session.

Prefer concrete terms such as `timer`, `Screen Time`, `blocked apps`, `allowed apps`, `sound`, `break`, `first page`, `first paragraph`, and `first problem`.

Do not use internal labels such as `setup-choice fatigue`, `focus tax`, `attention leak`, or `choices still alive` in visible copy.

## Why env fits

env saves timer, app-blocking, allowed-app, sound, break, cycle, and related settings as reusable environments. The student can run the relevant prepared environment without rebuilding those values at the starting moment.

## Available evidence

- Verified product capability: setup values can be saved in reusable environments and run again.
- Desk-research signal: students describe intending to study while continuing to use their phones.
- No direct env-user evidence currently proves that repeated setup choices deplete attention or delay study starts frequently.

## Do not exaggerate

- Do not say each choice measurably lowers concentration unless evidence is added.
- Do not guarantee that env creates focus, prevents every distraction, or improves grades.
- Do not claim that env solves motivation, chooses the work, or performs it.
- Do not present desk-research language as env-user testimony.
