# Message MSG-002 v2: Focus setup breaks across too many tools

**Status:** testing
**Version:** 2

## Target user and situation

A U.S. college student has decided to study, then moves between a timer, Screen Time, a sound app, and the Home Screen before the real work begins.

## Current belief

Moving through several focus tools is simply the normal route into a productive study session. Opening each tool only takes a moment, so the route itself does not matter.

## Current behavior

The student opens and configures separate focus tools while repeatedly returning to app lists, the Home Screen, or other surfaces where unrelated apps remain available.

## Intended belief shift

Every transition between setup tools creates another opening to leave the work the student already intended to begin.

The frequency and impact of this behavior are message hypotheses, not established user facts.

## Core perspective

The useful phone tools do not need to disappear. Prepare their recurring settings as one environment so the study session can begin without taking the long route through separate setup tools.

## Persuasion mechanism

1. Establish the literal route through separate setup tools and keep the intended work untouched during it.
2. Introduce unrelated app access as a plausible interruption, not an inevitable outcome.
3. Contrast that route with running one prepared environment.
4. Return attention to the intended study action rather than making env the endpoint.

## Reader resistance to address

- env might become one more app to configure.
- Opening the same tools manually is already easy.
- Combining the setup might remove tools needed for study.
- One setup cannot fit every session.

## Eligible situations

- A timer is set before leaving to configure Screen Time.
- The student returns to the Home Screen between the timer and sound app.
- An unrelated feed remains available while moving between app blocking and focus sound.
- The setup is completed while the first reading page still waits.

These are semantic examples of the message, not prescribed copy. The selected format's `copywriting.md` owns wording, hook, progression, tone, and product reveal.

## Why env fits

env runs prepared timer, Screen Time, allowed-app, sound, break, and related settings together as one reusable focus environment. Necessary apps can remain allowed. Widgets can optionally shorten entry into that environment but are not the core mechanism.

## Available evidence

- Verified product capability: the settings can be prepared and run together while necessary apps remain available.
- Low-confidence desk-research language describes students entering unrelated phone use near the beginning of intended study.
- No direct evidence currently establishes how often the target audience uses this combination of tools or how often transitions between them cause off-task app use.

## Do not exaggerate

- Do not say every setup transition causes distraction.
- Do not claim that the phone itself must disappear.
- Do not guarantee focus or complete distraction prevention.
- Do not claim env replaces Screen Time.
